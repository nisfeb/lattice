# Testing & CI Plan — `%lattice` desk + KMP app

> Goal: comprehensive, layered automated tests that run fast locally during
> development and gate PRs in CI, plus a heavier end-to-end suite that runs the
> real ship(s) on a schedule. Covers both halves of the repo: the Hoon
> `%lattice` desk (`desk/`) and the Compose Multiplatform app (`app/`).

## Principles

1. **Test pyramid.** Many fast pure-logic unit tests; fewer hermetic
   integration tests (mocked HTTP); a small number of UI tests; a thin,
   slow end-to-end layer against a real fakezod.
2. **PRs gate on fast, hermetic tests.** Anything needing a booted ship or a
   GUI window runs nightly or on-demand, never blocking every push — except a
   single bounded desk-unit job once it's stable.
3. **Pin the wire contracts.** The app↔desk HTTP/poke/scry shapes we
   reverse-engineered (fetch/list/save/delete, `settings-event` put-entry,
   `/~/scry/settings/...`) are encoded as explicit assertions, so a future
   change on either side that breaks the contract fails a test.
4. **Make logic testable.** Pure logic currently buried in `@Composable`s and
   inside the agent's `|%` core gets extracted so it can be unit-tested without
   a UI or a ship.

## Current state (baseline)

- App: `commonTest` wired with `kotlin("test")`; `GemtextTest`, `VimEngineTest`
  (~15 tests). Targets: `androidTarget` + `jvm("desktop")`.
- Desk: **no** automated tests; verification has been manual (curl + MCP
  against running fakezods `~zod`/`~tyr`).
- No `.github/workflows`. Dev ships are driven via an MCP server
  (`scripts/mcp-zod.sh`); there is no `urbit` binary on PATH (CI must fetch it).

## Layer 1 — Kotlin pure unit tests (`commonTest`, multiplatform)

Fast, deterministic, no I/O. Run on every target. Highest ROI; do first.

| Target | File | Cases |
|---|---|---|
| `gemtext/GemtextParser` | `GemtextTest` (exists) | extend: pre-blocks, quotes, bullets, headings levels, blank lines, CRLF |
| `gemtext/UrbUrl` | `UrbUrlTest` (new) | parse `urb://~ship/a/b`, bare ship, query; **resolve** relative (`b`, `./b`, `../b`, `/abs`) against a base; reject non-urb |
| `editor/VimEngine` | `VimEngineTest` (exists) | extend: `e`/`b` across lines, `c`+motion, `3dd`/`5j` counts, `dd` at last line, visual **linewise**, paste charwise vs linewise, undo stack depth, `:wq` signals |
| `theme/ColorHex` | `ColorHexTest` (new) | `#RRGGBB`/`#AARRGGBB`/no-`#`/bad → null; `toHex` round-trip; case-insensitivity |
| `theme/ThemeSettings` | `ThemeSettingsTest` (new) | color getters fall back on bad hex; presets parse; `vimMode` default false |
| `theme/SavedTheme` + repo serialization | `ThemeSerializationTest` (new) | `List<SavedTheme>` JSON encode/decode round-trip; unknown-keys tolerance |
| `ui/listDir` (FileTree) | `FileTreeTest` (new) | folder/file split at root and nested dir; `..` parent; dedup folders; sorting |
| `browser` URL helpers (extracted, see refactors) | `UrlPathsTest` (new) | `defaultDest("urb://~tyr/notes/x")=="notes/x"`, home→ship; `editPathFor` own-ship→path, home→`"index"`, remote→null; `tabTitle` |
| `ui/overflowSplit` (extracted) | `OverflowTest` (new) | given width/unit/count → (inline, overflow) counts; min-width honored; everything inline when wide |

Tooling: `kotlin.test` assertions; `kotlinx-coroutines-test` for any `suspend`
helpers. Coverage target: **≥85%** on `gemtext`, `editor`, `theme`, and the
extracted `browser`/`ui` helper files.

## Layer 2 — Kotlin client integration (`desktopTest` + MockWebServer)

JVM-only (OkHttp). Spin up `okhttp3.mockwebserver.MockWebServer`, point a real
`UrbitSession` at its URL, drive the real client code, and assert **both** the
outgoing request (method/path/query/body/headers) **and** response parsing.
These pin the wire contracts.

- **`UrbitSessionTest`**: `login` → `POST /~/login` body `password=<code>`;
  captures `Set-Cookie: urbauth-~ship`; subsequent requests carry the cookie;
  `tryRestore` reuses a `SavedSession`; `logout` clears it. 401 → failure.
- **`LatticeClientTest`**:
  - `fetch` → `GET /apps/lattice/fetch?url=<urlencoded>`; parses
    `{"mark","body"}`; `{"error":...}` → `Result.failure`.
  - `list` → `GET /apps/lattice/list`; parses + sorts `{"files":[...]}`.
  - `save` → `POST /apps/lattice/save?path=<p>`; **request body == content**,
    `text/plain`; non-2xx → failure.
  - `delete` → `POST /apps/lattice/delete?path=<p>`.
- **`SettingsClientTest`**:
  - `putEntry` → `PUT /~/channel/<id>` with JSON
    `[{id,action:"poke",ship,app:"settings",mark:"settings-event",
    json:{"put-entry":{desk,bucket-key,entry-key,value}}}]`; `value` is the
    JSON-stringified payload.
  - `readEntry` → `GET /~/scry/settings/desk/<desk>.json`; parses
    `{"desk":{bucket:{entry:value}}}`; absent → null.
- **`ThemeRepositoryTest`**: `push` then `pull` round-trips a named-theme list
  through MockWebServer (and via a fake `SettingsClient`).

Fixtures: capture real response bodies (we already have them from curl) under
`desktopTest/resources/fixtures/` so mocks stay faithful.

## Layer 3 — Kotlin Compose UI tests (`desktopTest`, headless)

`@OptIn(ExperimentalTestApi) runComposeUiTest { setContent { … } }` — renders
offscreen via Skiko, no real window. Inject **fakes** (the stores are already
interfaces; `LatticeClient` driven by MockWebServer or a thin fake). Keep this
set small and high-value:

- **WorkspaceScreen**: opening a file adds a tab; the `⋮` menu shows
  Duplicate/Move/Delete; Delete shows a confirmation dialog; folder row drills
  in / `..` goes up; Save button enables on edit.
- **BrowserScreen**: `+`/Ctrl+T adds a tab; close (×)/Ctrl+W removes it;
  narrowing the test window moves buttons into the `⋮` overflow; address Go
  navigates.
- **SettingsScreen**: vim toggle flips `vimMode`; editing a color hex updates
  the live preview swatch; preset chip applies colors but preserves vim.
- **PlainEditor / VimEditor**: typing updates text + dirty; Ctrl+S / `:w`
  fire the save callback; vim `o`/`dd`/`x` mutate as expected through the UI.

Risk: Skiko headless on Linux occasionally needs `xvfb-run`; wrap the CI task
if needed. Keep these resilient (`waitForIdle`, no real delays).

## Layer 4 — Hoon desk unit tests (`/lib/test`)

**Refactor first:** extract the agent's pure helpers into `desk/lib/lattice.hoon`
(imported by `app/lattice.hoon`), then test the lib — pure gates don't need a
running agent.

`desk/tests/lib/lattice.hoon` (run by the standard `-test` runner):
- `++ test-parse-urb-url` — `urb://~zod/a/b`→`[~zod /a/b]`; bare ship→`[~zod /]`;
  non-urb→`~`.
- `++ test-lib-path` — `'notes/idea'`→`/lib/notes/idea/gmi`.
- `++ test-req-action` — last path segment from a request URL.
- `++ test-query-param` — extract + `de-urlt` decode.
- `++ test-generate-index` — set of paths → gemtext `=>` links (sorted).
- `++ test-keen-path` — spur → `/g/x/1/lattice//1/<spur>`.
- `++ test-mark-and-body` — `(mark body)` → expected JSON.

`desk/tests/mar/gmi.hoon` (if practical): `gmi` mark `grab`/`grow` round-trip
(noun↔text), and `grad %txt` conversion.

Agent lifecycle (`on-poke`/`on-arvo`/`on-watch`) is **not** unit-tested in-kernel
(low ROL, high effort); it's covered by Layer 5.

## Layer 5 — End-to-end (real fakezod)

The real contract guardian: app's client code (or curl) against the actual
running desk, including Clay publish and cross-ship `%keen`.

- **Dev (fast inner loop):** reuse the existing MCP/curl flow against the
  already-running `~zod`. Add `scripts/e2e.sh` that: logs in, `list`/`save`/
  `fetch`/`delete` round-trip, asserts publish (`%grow`) + cull, and — if a 2nd
  ship is reachable — a cross-ship `fetch urb://~tyr/...`. Pure bash + curl +
  `jq`; asserts on JSON. (Mirrors `docs/lattice-integration-test.md`.)
- **CI (nightly):** a job that fetches a pinned `urbit` binary + cached solid
  pill, boots a fresh fakezod `-d`, mounts + commits `desk/`, installs
  `%lattice`, then runs `scripts/e2e.sh` against it. Cross-ship variant boots
  two piers. Slow (minutes) and the flakiest layer → **schedule + manual
  dispatch only**, not PR-gating.

## Testability refactors (prerequisite work)

App:
- Extract from `BrowserScreen`: `defaultDest(url)`, `editPathFor(url, ownPrefix)`,
  and `tabTitle(url)` → a pure `browser/UrlPaths.kt` (top-level funcs).
- Extract the responsive split into `fun overflowSplit(widthDp, unitDp, reservedDp, count): IntRange`-ish pure helper used by the bar.
- `listDir` is already a pure top-level fun (good).
- Confirm `UrbitSession(OkHttpClient, SessionStore)` + `login(baseUrl, code)` let
  tests point at a MockWebServer URL (they do).

Desk:
- New `desk/lib/lattice.hoon` exporting the pure helpers above; `app/lattice.hoon`
  switches to `/+  *lattice` (or `/+ lattice`) and calls them. Scry-dependent
  helpers (`list-gmi`, `walk-dir`, `file-hash`) stay in the agent.

## Tooling / Gradle changes

`libs.versions.toml` add: `mockwebserver` (= okhttp 4.12.0),
`kotlinx-coroutines-test`, Compose `uiTest` artifacts, and the **Kover** plugin.

`composeApp/build.gradle.kts`:
```kotlin
sourceSets {
  commonTest.dependencies {
    implementation(kotlin("test"))
    implementation(libs.kotlinx.coroutines.test)
    @OptIn(ExperimentalComposeLibrary::class)
    implementation(compose.uiTest)
  }
  val desktopTest by getting {
    dependencies {
      implementation(libs.mockwebserver)
      implementation(compose.desktop.currentOs)        // UI test runtime
      implementation(compose.desktop.uiTestJUnit4)
    }
  }
}
```
Add Kover for coverage reports (`./gradlew koverHtmlReport`, `koverVerify` with
per-package thresholds).

## CI design (GitHub Actions)

**`.github/workflows/ci.yml`** — on `pull_request` + push to `main`; gating:
- `app`: `actions/setup-java@v4` (Temurin 21) → `android-actions/setup-android@v3`
  (+accept licenses, needed because `androidTarget` makes Gradle require the SDK)
  → `gradle/actions/setup-gradle@v4` (cache) → run:
  - `./gradlew :composeApp:compileKotlinDesktop :composeApp:compileDebugKotlinAndroid`
  - `./gradlew :composeApp:desktopTest` (Layers 1–3; wrap in `xvfb-run` if UI
    tests need it) → upload Kover report + test results.
- `desk-unit` (promote to gating once stable; nightly until then): fetch/cache
  `urbit` + pill, boot fakezod, `-test %/tests` over `desk/tests`, parse
  results. (Hoon tests need a runtime; this is the only "slow" gating job.)

**`.github/workflows/nightly.yml`** — `schedule` (cron) + `workflow_dispatch`:
- `e2e`: boot fakezod(s), install desk, run `scripts/e2e.sh` (single- and
  cross-ship). Allowed to be slow; retries; artifacts = ship logs on failure.

Caching: Gradle (`setup-gradle`), Android SDK, urbit binary + pill (`actions/cache`).

## Local dev workflow

- Fast: `./gradlew :composeApp:desktopTest` (all JVM tests) — the inner loop.
- Pure-only quick: `./gradlew :composeApp:desktopTest --tests '*Test'` filters.
- Coverage: `./gradlew koverHtmlReport`.
- Desk unit (needs a ship): `scripts/desk-test.sh` — commits `desk/` to the
  running `~zod` via MCP and runs `-test`.
- E2E: `scripts/e2e.sh [--cross-ship]` against the running fakezod(s).
- Optional `Makefile`/`justfile`: `make test` (gradle), `make test-desk`,
  `make e2e`, `make ci` (everything).

## Fixtures

- `desktopTest/resources/fixtures/`: real captured JSON
  (`fetch.json`, `list.json`, `settings-desk.json`) + sample `.gmi` documents
  for the parser. Reuse `desk/lib/*.gmi` where possible.

## Coverage & reporting

- Kover, fail-under thresholds on logic packages (85% pure, 70% overall).
- Surface the report as a CI artifact + PR summary.

## Phased rollout

1. **Phase 1 (fast win):** Layer 1 extractions + tests (UrbUrl, ColorHex,
   ThemeSettings/serialization, listDir, UrlPaths, overflowSplit, VimEngine
   expansion) + Kover. ~½–1 day.
2. **Phase 2:** Layer 2 MockWebServer client tests + fixtures + `ci.yml` `app`
   job (compile both targets + desktopTest). ~1 day. **← PRs start gating here.**
3. **Phase 3:** Layer 3 Compose UI tests (focused set). ~1 day.
4. **Phase 4:** Layer 4 `/lib/lattice.hoon` extraction + Hoon unit tests +
   `desk-unit` CI job. ~1 day.
5. **Phase 5:** Layer 5 `scripts/e2e.sh` + `nightly.yml`. ~1 day.

## Status (implemented)

All five phases landed:
- **L1–L3** run in `ci.yml` (PR-gating): `:composeApp:desktopTest` (67 tests —
  pure unit, MockWebServer integration, headless Compose UI under xvfb) plus
  desktop+android compile and a Kover report artifact.
- **L4**: pure helpers extracted to `desk/lib/lattice.hoon`; `desk/tests/lib/
  lattice.hoon` (7 arms) green via the `run-tests` MCP tool; `/lib/test`
  vendored into the desk; `scripts/desk-test.sh` runs them in dev.
- **L5**: `scripts/e2e.sh` (login/list/save/fetch/delete + cross-ship) passes
  against the live ship; `nightly.yml` runs it.

**Realized CI deviation:** the nightly suite targets a **persistent test ship
via secrets** (`LATTICE_URL`/`LATTICE_CODE`, optional `LATTICE_PEER_SHIP`,
`LATTICE_MCP_COOKIE`) and skips cleanly when unset — rather than booting a
fakezod inside the runner. Booting + driving a ship in CI needs the `urbit`
binary and the dev's `%mcp` control path, neither of which is hermetic on a
stock runner; hitting a maintained ship is simpler and reliable. Booting a
throwaway fakezod remains a future option if a self-hosted runner is added.

## Open decisions

- **Does `desk-unit` gate PRs or run nightly?** Depends on tolerance for a
  ~2-min fakezod boot in CI. Recommend: nightly first, promote to gating once
  the boot+`-test` harness is reliable.
- **Pin urbit version/pill** for CI reproducibility (record in the workflow).
- **ktlint/detekt** for style — optional Phase 2 add-on.
